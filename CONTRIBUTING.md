# Contributing to OpenClaw Delegation Patterns

Thank you for your interest in improving delegation patterns for OpenClaw!

## How to Contribute

### 1. Reporting Issues

Found a bug or have a suggestion? Open an issue with:
- Clear description of the problem
- Steps to reproduce (if applicable)
- Expected vs actual behavior
- Environment details (OpenClaw version, model used)

### 2. Submitting Improvements

#### Documentation Improvements
1. Fork the repository
2. Create a feature branch: `git checkout -b docs/improve-clarity`
3. Make your changes
4. Run spell check before committing
5. Submit a pull request

#### New Patterns or Examples

When adding new patterns:
1. Follow the existing file structure
2. Include working code examples
3. Document the problem the pattern solves
4. Test the pattern before submitting

### 3. Pull Request Guidelines

**PR Title Format**: `[type]: Brief description`

Types:
- `docs:` - Documentation changes
- `fix:` - Bug fixes
- `feat:` - New features/patterns
- `refactor:` - Code improvements

**PR Description Should Include**:
- What the change does
- Why it's needed
- How to test it

### 4. Code Style

- Use clean, readable markdown
- Include code blocks with syntax highlighting
- Keep lines under 100 characters when possible
- Use American English spelling

### 5. Review Process

1. PR submitted
2. Community review (2-3 days)
3. Address feedback
4. Merge approved PRs

## Getting Help

- Open an issue for questions
- Check existing issues before creating new ones
- Be respectful and constructive

## Recognition

Contributors will be acknowledged in the README and CONTRIBUTORS file.
